### Projet de A à Z
* Creer un répertoire, avec 2 sous-répertoires
  * `projet`, `projet/app`, `projet/init_db`
* Creer votre application (répertoire `app`)
* Creer le conteneur de base de données
  * l'option -v pour instancier la bd (table, utilisateurs...) (répertoire `init_db`)
  * l'option -v pour créer la bd sur le disque de l'ordinateur (à l'extérieur `../database`)
* Fichier dockerfile dans le répertoire `app`
* Fichier docker-config dans le répertoire `projet`

### Une bonne note sur le rapport
* Contenant: 
    * En PDF
    * Antidote (80$ le mieux investi de votre vie)
    * Interligne 1.5
    * Enlever les pronoms personnels (Je, Nous/on, notre...)
    * Les phrases : 2 lignes max

---

### Une bonne note sur le rapport
* Contenu :
    * Introduction
      * Présentation du problème
      * Présenter les sections
    * Développement
      * Section = créer un flow
      * Intro et conclusion de sous-section!
      * Critères de correction = nom de sous-section
      * Justifiez TOUS vos choix de design
    * Conclusion
      * Avez-vous résolu le problème? Comment?
